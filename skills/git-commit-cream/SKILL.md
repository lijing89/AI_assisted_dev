# Git Commit Cream

此技能用于生成高质量、简洁的中文 Git 提交信息，并根据特定规则过滤变动。

## 核心规则

1.  **忽略特定目录**：
    *   在分析代码变动时，必须**完全忽略** `ios/` 和 `android/` 目录下的所有文件变动。
    *   如果变动仅包含这些目录下的文件，应提示用户“无有效变动（已忽略移动端文件）”。

2.  **语言要求**：
    *   必须使用**中文**。

3.  **内容要求**：
    *   **简洁**：直击要点，避免冗余描述。
    *   **基于事实**：根据实际修改的文件和内容生成信息。

4.  **格式规范**：
    *   推荐使用 `<type>: <subject>` 格式。
    *   Type 参考：`feat` (新功能), `fix` (修复), `docs` (文档), `style` (格式), `refactor` (重构), `perf` (性能), `test` (测试), `chore` (构建/工具), `ci` (集成)。

## 使用流程

1.  获取当前的 git status 或 diff 信息。
2.  **过滤步骤**：移除所有路径以 `ios/` 或 `android/` 开头的条目。
3.  **生成步骤**：
    *   分析剩余文件的改动意图。
    *   生成一句简洁的中文描述作为 subject。
    *   (可选) 如果变动较多，可列出要点。

## 示例

**场景 1**
*   变动文件：`src/api/user.js` (修改登录逻辑), `ios/App/Info.plist` (版本号)
*   **输出**：`fix: 优化用户登录逻辑` (注意：ios 文件被忽略)

**场景 2**
*   变动文件：`android/build.gradle`
*   **输出**：无有效变动（已忽略移动端文件）。
